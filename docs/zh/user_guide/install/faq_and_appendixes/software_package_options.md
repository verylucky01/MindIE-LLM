# 软件包参数说明

软件包支持根据命令行完成一键式安装，各个命令之间可以配合使用，用户根据安装需要选择对应参数完成安装，所有参数都是可选参数。

安装命令格式：./_软件包名_.run\[options\]

详细参数请参见[表1](#table1)。

**表 1**  软件包支持的参数说明 <a id="table1"></a>

|参数|说明|
|--|--|
|--help \| -h|查询帮助信息。|
|--info|查询软件包构建信息。|
|--list|查询软件包文件列表。|
|--check|检查软件包的一致性和完整性。|
|--quiet | -q|静默安装，跳过交互式信息。|
|--extract=\<path\>|解压软件包中文件到指定目录。|
|--install-path=\<path\>|该参数仅支持绝对路径作为输入，不支持相对路径。<br>可在如下目录查看是否存在该文件：<ul><li>root用户：/etc/Ascend</li><li>非root用户：/home/*{当前用户名}*/Ascend</li></ul>若不指定，将安装到默认路径下：<ul><li>若使用root用户安装，默认安装路径为：/usr/local/Ascend。</li><li>若使用非root用户安装，则默认安装路径为：*/*home*/{当前用户名}*/Ascend。</li></ul>若通过该参数指定了安装目录，运行用户需要对指定的安装路径有可读写权限。|
|--install|安装软件包。后面可以指定安装路径--install-path=\<path\>，也可以不指定安装路径，直接安装到默认路径下。|
|--upgrade|升级已安装的软件，支持在低版本升级至高版本场景下使用。后面可以指定升级路径--install-path=\<path\>，也可以不指定升级路径，直接升级默认路径。<br>使用该参数升级后，新版本将继承低版本Server中config.json文件的配置信息。|
|--uninstall|卸载已安装的软件。后面可以指定卸载路径--install-path=\<path\>。|
